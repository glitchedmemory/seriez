## 1. 서비스명 (제안)

미정. 아래 후보 중 선택하거나 X님이 직접 정하실 수 있습니다:

- **Mova** (Movie + OVA) — 짧고 기억하기 쉬움, 도메인 저렴
- **Cinevault** — 시네마 + 볼트(보관소). 시청 기록 저장의 의미
- **Showstack** — 쌓아둔 쇼들. 리스트 쌓는 느낌
- **Flicklist** — 영화(flick) + 리스트
- **Watchpedia** — 시청 백과사전

## 3. 전체 유저 여정 (End-to-End)

### Step 1: 첫 방문 → 가입 (30초)

```
사용자가 사이트 접속
→ 랜딩 페이지: "당신의 시청 인생을 기록하세요" + 인기 포스터 모자이크 배경
→ [Google로 시작하기] / [GitHub로 시작하기] / [이메일로 가입]
→ 가입 즉시 온보딩 화면으로
```

**따로 비밀번호 만들 필요 없음.** Google/GitHub OAuth로 1클릭 가입. 이메일 가입 시에도 비밀번호만 입력하면 끝.

### Step 2: 온보딩 (90초 — 취향 파악)

```
[1/4] 어떤 장르를 좋아하세요? (중복 선택 가능)
  🎬 Action  🧠 Sci-Fi  😂 Comedy  😱 Horror
  💕 Romance  🎭 Drama  🎞️ Thriller  🌏 Anime
  [건너뛰기]

[2/4] 최근에 재밌게 본 작품 3개만 알려주세요
  [🔍 검색해서 추가...]
  [건너뛰기]

[3/4] 한 번에 주로 몇 편 보세요?
  ○ 가끔 생각날 때 (주 1~2편)
  ○ 꾸준히 보는 편 (주 3~5편)
  ○ 진짜 많이 봐요 (거의 매일)

[4/4] 어떤 걸 더 추천받고 싶으세요?
  ○ 영화 위주
  ○ TV쇼 위주
  ○ 애니 위주
  ○ 골고루 다

→ [시작하기]
```

**핵심:** 4단계 모두 건너뛰기 가능. 취향 데이터가 없어도 서비스 이용에 지장 없음. 다만 입력할수록 추천 정확도가 올라감.

### Step 3: 홈 화면 (매일 방문하는 곳)

```
상단: [🔍 제목, 감독, 배우 검색...]
      ────────────────────────────────
      🎯 For You
      지금 OOO님께 추천하는 작품 6개 (가로 스크롤)
      → 온보딩에서 입력한 취향 기반 + TMDB Similar API
      → "왜 추천했는지" 한 줄 근거 표시
        예: "Dune: Part Two를 좋아하셨기 때문에"

      🔥 Trending This Week
      전 세계에서 가장 많이 기록된 작품 10개

      👥 친구 활동
      (팔로우한 친구가 없으면 "친구를 찾아보세요" CTA)
      지민님이 Inception에 ★★★★½
      수현님이 Severance S2를 Completed
      태호님이 위시리스트에 Oppenheimer 추가

하단 탭바: [🏠 Home] [🔍 Search] [📚 Library] [👤 Profile]
```

### Step 4: 작품 검색 → 상세 페이지 → 기록

```
검색어: "interstellar"
→ 실시간 검색 결과 (TMDB + AniList 동시 쿼리)
→ 영화 / TV / 애니 탭으로 필터링

Interstellar (2014) 클릭
→ ┌─────────────────────────┐
   │      포스터 (큰 버전)    │
   │                          │
   │ Interstellar             │
   │ 2014 · SF · 2h 49m      │
   │ ★ 4.7/5 (12,384명)     │
   │                          │
   │ [★ 평점 남기기]          │
   │ [📋 리스트에 추가]       │
   │ [상태: 안 봄 ▼]         │  ← 드롭다운: Plan to Watch
   │                          │     / Watching / Completed
   └─────────────────────────┘

   시놉시스
   지구가 황폐화된 미래, 전직 NASA 조종사 쿠퍼가
   인류의 새 보금자리를 찾아 우주로 떠난다.

   🎬 감독: Christopher Nolan
   👥 출연: Matthew McConaughey, Anne Hathaway...

   🔗 비슷한 작품
   [Inception] [The Martian] [Gravity] [Arrival]

   📝 리뷰 (48)
   ┌─────────────────────────┐
   │ ⭐⭐⭐⭐⭐ user123       │
```

### Step 5: 별점 남기기

```
[★ 평점 남기기] 탭
→ 별 5개 UI 등장
→ 터치 드래그: ★★★★☆ (4.0)
→ 별 아래 "마음에 들었어요!" 같은 감정 태그 선택 가능
  😍 최고  👍 좋아요  😐 무난  👎 별로  💀 최악
→ (선택) 한 줄 리뷰 작성
→ [저장]
```

**별점 기록 즉시 효과:**
- 추천 알고리즘에 반영
- 친구 활동 피드에 표시
- 프로필 통계 업데이트

### Step 6: 시청 상태 트래킹

사용자는 각 작품을 5가지 상태로 관리:

| 상태 | 의미 | 예시 |
|------|------|------|
| **Plan to Watch** | 보고 싶은 목록 | "언젠가 봐야지" 리스트 |
| **Watching** | 지금 보는 중 | TV쇼 시즌 진행 중, 에피소드 추적 |
| **Completed** | 다 봄 | 평점 + 리뷰 남기기 권장 |
| **On Hold** | 잠시 중단 | 시즌 끝나서 기다리는 중 |
| **Dropped** | 포기 | "보다가 접었다" |

**TV쇼 특별 기능:**
- 에피소드 단위 진행도 체크 (S2 E5 / 12)
- 다음 에피소드 방영일 알림
- "다 본 시즌" 자동 Completed 처리 여부 설정

## 4. 추천 시스템 작동 방식

### 4.1 추천 데이터 소스

| 소스 | 제공 정보 | 사용처 |
|------|----------|--------|
| **내 시청 기록** | 본 작품, 평점, 좋아하는 장르 | 1차 추천 기반 |
| **내 평점 패턴** | 어떤 장르/감독에 높은 점수를 줬는지 | 가중치 계산 |
| **TMDB Similar API** | 특정 영화와 유사한 작품 목록 | "Inception 좋아하면 이것도" |
| **TMDB Recommendations** | TMDB 자체 추천 알고리즘 | 보조 추천 |
| **글로벌 트렌드** | 전 유저의 시청·평점 데이터 | 인기 급상승 섹션 |

### 4.2 추천 로직 (단계별)

```
[STEP 1] 사용자 프로필 구축
→ 시청한 작품들의 장르 분포: Sci-Fi 40%, Drama 30%, Action 20%, 기타 10%
→ 선호 감독 Top 5: Christopher Nolan, Denis Villeneuve...
→ 평균 평점: 3.8/5
→ 신뢰 가능한 유사 유저 20명 찾기 (비슷한 장르 분포 + 평점 패턴)

[STEP 2] 후보군 생성
→ 선호 장르에서 평점 높은 미시청 작품 (TMDB Discover API)
→ 선호 감독의 다른 작품
→ 내가 높게 평가한 작품의 Similar 작품 (TMDB)
→ 유사 유저들이 높게 평가한 작품

[STEP 3] 필터링 + 순위화
→ 이미 본 작품 제외
→ 이미 Dropped/On Hold 제외 (다시 추천 안 함)
→ 가중치 계산:
  - 장르 부합도: 40%
  - Similarity 점수: 25%
  - 유사 유저 평점: 20%
  - 글로벌 평점: 15%
→ 상위 20개 추출

[STEP 4] 이유 생성
→ 각 추천에 "왜"를 한 줄로 표시
→ 예: "Inception을 ★★★★★ 주셨기 때문에"
→     "SF 장르를 가장 좋아하시기 때문에"
→     "비슷한 취향의 유저들이 ★4.5를 줬어요"
```

### 4.3 추천 표시 위치

- **홈 화면 — 🎯 For You**: 상위 6개 (매일 갱신)
- **작품 상세 — 🔗 비슷한 작품**: 해당 작품 기준 6개
- **프로필 — 📊 Based on Your Top Genres**: 장르별 추천
- **검색 결과 — 💡 You Might Like**: 검색 맥락 기반 3개

## 5. 소셜 기능 작동 방식

### 5.1 친구 팔로우
- 검색으로 유저 찾기 (username, 이메일)
- 팔로우는 단방향 (인스타그램처럼)
- 팔로우 수락 불필요 (기본 공개 프로필)

### 5.2 활동 피드
```
메인 화면에서 친구들의 최근 활동이 시간순으로 표시:

[40분 전] 지민님이 Interstellar ★★★★½
         "다섯 번째인데도 눈물나네..."

[2시간 전] 태호님이 Severance S2 — Completed
            시즌 2까지 전부 시청 완료!

[5시간 전] 수현님이 Wishlist에 Dune 3 추가
            개봉 전부터 찜!

[어제]     영수님이 리스트 "2026 Best Sci-Fi" 생성
            12작품 · 5명 저장
```

### 5.3 리스트
- **Watchlist** (기본 제공): 보고 싶은 작품 모음
- **Favorites** (기본 제공): 인생 작품 Top 10
- **커스텀 리스트** (무제한): "주말에 볼 영화 5선", "크리스토퍼 놀란 정주행"
- 리스트는 공개/비공개 설정 가능
- 다른 사람의 리스트를 내 리스트로 복제 가능

## 8. 기술적 작동 방식

### 8.1 데이터 흐름

```
[사용자] → 검색어 입력
    → Next.js API Route → TMDB API / AniList GraphQL 호출
    → 결과 렌더링

[사용자] → 상태 변경 (Watching → Completed)
    → Supabase DB에 저장
    → 추천 캐시 무효화 (다음 추천에 반영)

[시스템] → 매일 자정 추천 재계산
    → Supabase Edge Function
    → 각 유저의 시청 기록 + 평점 바탕으로 후보군 생성
    → TMDB API로 정보 보강
    → 결과 캐싱 (24시간)
```

### 8.2 API 호출 최적화

- TMDB 이미지는 Next.js Image 컴포넌트로 최적화 (자동 리사이징, WebP 변환)
- 검색 결과는 5분간 Redis 캐싱 (Vercel KV)
- 인기 작품은 1시간 갱신
- 유저별 추천은 24시간 계산 → 당일 내내 재사용

### 8.3 PWA 작동 방식

- 최초 방문 시 앱 셸(HTML+CSS+JS) 캐싱 → 다음 방문 즉시 로딩
- 오프라인 상태에서도 이전 검색 결과, 내 Library 열람 가능
- 인터넷 연결 복구 시 백그라운드 동기화
- 모바일에서 "홈 화면에 추가" → 네이티브 앱처럼 실행 (상단 URL 바 없음)

## 9. 비즈니스 모델

| 단계 | 가격 | 제공 기능 |
|------|------|----------|
| **Free** | $0 | 무제한 트래킹, 기본 통계, 광고 제거, 720p 포스터 |
| **Pro** | $2.99/mo | 상세 통계, 연간 리포트, HD 포스터, 스트리밍 연동 알림, 첫 30일 무료 |

- 무료로도 핵심 가치(트래킹+추천)를 충분히 제공

- Letterboxd Pro($19/yr ≈ $1.58/mo)보다 살짝 높게, Trakt VIP($5/mo)보다 낮게
- 연간 결제 시 20% 할인 ($28.70/yr ≈ $2.39/mo)

## 10. 경쟁사와의 차이 (요약)

| 요소 | Letterboxd | Trakt | AniList | 우리 서비스 |
|------|-----------|-------|---------|------------|
| 영화 | ✅ | ✅ | ❌ | ✅ |
| TV쇼 | ❌ | ✅ | ❌ | ✅ |
| 애니 | ❌ | △ | ✅ | ✅ |
| UI 디자인 | ★★★★★ | ★★★ | ★★★★ | ★★★★★ (타겟) |
| 개인화 추천 | ❌ | △ | △ | ✅ 핵심 기능 |
| 추천 이유 표시 | ❌ | ❌ | ❌ | ✅ |
| PWA | ❌ | ❌ | ❌ | ✅ |
| 무료 제공량 | 많음 | 적음(VIP 유도) | 전부 무료 | 많음 |
| 다국어 지원 | ❌ | ❌ | ❌ | 향후 계획 |

**핵심 차별점 하나:**
> Letterboxd는 "기록"에 집중하고, Trakt는 "추적 자동화"에 집중하지만, 우리는 **"기록을 바탕으로 한 똑똑한 추천"** 이 핵심입니다. "왜 이걸 추천했는지" 이유를 항상 보여주는 건 아무도 안 하고 있습니다.
