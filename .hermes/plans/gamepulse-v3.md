# 🏀 GamePulse — 기획서 v3 (옵션 B: AI 질문 기반 라이브 스포츠 앱)

## 1. 컨셉 (한 줄)

**"경기 보면서 궁금한 거 물어보면 바로 답해주는 앱."**

탭·메뉴 없음. 질문 하나로 모든 정보에 접근한다. 
경기 중 해설자가 던진 말, 트위터에서 본 글, 갑자기 궁금해진 선수 기록 — 
구글 갈 필요 없이 앱에 바로 묻는다.

---

## 2. 왜 지금 이걸 만들어야 하는가

### 2.1 기존 앱의 결정적 빈틈

| 경쟁사 | 강점 | 결정적 약점 |
|---|---|---|
| **theScore / ESPN** | 모든 정보 다 있음 | 4~5탭 탐색. "질문→답변" 구조 아님 |
| **StatMuse** | 자연어 스포츠 Q&A | **라이브 경기 데이터 없음. 과거 기록만.** |
| **Watchers.io** | AI 스포츠 어시스턴트 | **B2B 전용. 소비자 앱 아님.** |
| **Google** | 모든 걸 섞어서 답변 | 타이핑 필요. 맥락 기억 못 함. 느림. |
| **SofaScore** | 고급 스탯·시각화 | Q&A 없음. 데이터 탐색만 |

### 2.2 시장의 빈 공간

**라이브 경기 중 실시간 데이터로 자연어 질문에 답하는 소비자용 앱 — 존재하지 않음.**

StatMuse가 가장 가까웠지만:
- "Brunson이 오늘 몇 득점이야?" → **답변 불가** (게임 종료 후에만 가능)
- "지금 Spurs가 왜 지고 있어?" → **답변 불가** (분석 없음)
- "Wembanyama 평소보다 블록 잘하고 있어?" → **답변 불가** (실시간 비교 없음)

Watchers는 B2B라서 일반 소비자가 쓸 수 없음.

**→ 이 틈새가 GamePulse의 자리다.**

## 3. 핵심 기능

### 3.1 AI 질문 — 자유 입력

```
사용자: "Brunson 오늘 몇 득점이야?"

GamePulse: 34득점 (12-18 FG, 5-7 3PT)
           시즌 평균 28.3pts 대비 +20%
           현재 Q4 4:12, Knicks 102-98 리드 중
```

```
사용자: "왜 Spurs가 지고 있어?"

GamePulse: 3가지 요인:
           ① 3점슛 6-22 (27.3%) — 시즌 평균 36.1% 대비 큰 하락
           ② 턴오버 18개 (NYK 9개 대비 2배)
           ③ Wembanyama 파울 트러블 — 4파울, 3쿼터 8분만 출전
```

```
사용자: "Wemby 평소보다 블록 잘하고 있어?"

GamePulse: 네. 현재 4블록, 시즌 평균 3.1개 대비 +29%.
           특히 3쿼터에만 3블록 — Brunson 상대 2회, Bridges 상대 1회.
```

### 3.2 자동 질문 카드 — 묻지 않아도 미리 답변

앱이 경기 데이터를 실시간으로 분석해서 **지금 궁금할 만한 질문을 예측 생성.**

```
🏀 NYK 102 — 98 SAS | Q4 4:12

🤔 지금 가장 궁금할 정보:

• Brunson 34pts로 오늘 최다득점
  → 커리어 플레이오프 3번째 30+ 경기
  
• Wemby 4블록, 근데 파울 4개
  → 3Q 8분만 출전. 파울 트러블 진행 중
  
• 3Q에 NYK 12-0 run 있었음
  → Spurs 3점슛 0-7 구간. 4분 무득점
  
• NYK -3.5 라인이 오프닝 -1.5에서 상승
  → 베팅 머니 78% Knicks 커버 방향
```

### 3.3 경기 타임라인 — 시계열로 모든 순간

경기 주요 이벤트를 타임라인으로 표시. 각 이벤트 클릭 시 AI가 설명.

```
Q1 ─────────────────────────────────────────────── Q4

[8:14 Q1] 🟢 Brunson 3PT      → "경기 첫 3점. NYK 8-2 리드"
[4:22 Q1] 🟡 Wemby Block #1   → "Brunson 레이업 블록"
[0:05 Q1] 🔴 Wemby Dunk       → "앨리웁. SAS 28-26 역전"
[9:31 Q2] ⚠️ Wemby PF #2     → "2파울. 벤치로. 2분 출전 제한"
[4:18 Q3] 🔥 NYK 12-0 run     → "[자세히] Spurs 4분 무득점"
[0:00 Q3] 🟢 Brunson 30pts    → "3Q까지 30득점. FG 10-15"
```

### 3.4 푸시 알림 — 질문을 미리 답변

백그라운드에서 AI가 경기 데이터 모니터링하다 중요 변화 감지 시 푸시.

```
🔔 Brunson 40득점 돌파!
   플레이오프 커리어 하이. FG 14-20 (70%).
   NYK 112-104, Q4 2:01

🔔 Wemby 파울 아웃!
   6파울 퇴장. 12pts 10reb 4blk.
   SAS 잔여 3:42, 8점 열세
```

### 3.5 음성 질문 — 게임 보면서 묻는다

"Hey GamePulse, Brunson 오늘 40점 넘었어?" 
→ 음성 답변 + 화면에 스탯 표시

## 4. 기술 아키텍처

### 4.1 데이터 파이프라인

```
SportsDataIO (live scores/stats)
    │
    ▼
Redis (실시간 캐시, 5초 TTL)
    │
    ▼
AI 엔진 (DeepSeek V4 Pro)
    ├── 질문 이해 (NLU)
    ├── 데이터 조회 (SQL/API)
    ├── 분석·비교·맥락 생성
    ├── 답변 생성 (NLG)
    └── 질문 카드 자동 생성
    │
    ▼
Supabase Realtime → 클라이언트
```

### 4.2 AI 엔진 상세

| 레이어 | 기능 | 기술 |
|---|---|---|
| **NLU** | 질문 의도 파악 + 엔티티 추출 ("Brunson"=선수, "오늘"=현재 경기, "득점"=pts) | LLM + few-shot |
| **Query** | 구조화된 데이터 조회로 변환 → SportsDataIO API 호출 | Function calling |
| **Context** | 시즌 평균·커리어 기록·상대 전적 등 비교 맥락 추가 | PostgreSQL |
| **NLG** | 자연어 답변 생성 | LLM (DeepSeek) |
| **Prediction** | "지금 궁금할 질문" 자동 생성 → 카드 프리렌더 | 크론잡 (60초) |

### 4.3 스택

| 레이어 | 기술 | 근거 |
|---|---|---|
| 프론트 | **Next.js 16 + PWA** | Seriez와 동일 기술. 네이티브 앱처럼 설치 가능 |
| DB | **Supabase** (PostgreSQL + Realtime) | 무료 티어. 실시간 구독으로 경기 업데이트 푸시 |
| 캐시 | **Redis** (Upstash 무료) | API 호출 중복 방지. 실시간 데이터 5초 TTL |
| AI | **DeepSeek V4 Pro** (기존) | 빠르고 저렴. Function calling 지원 |
| 데이터 | **SportsDataIO** (NBA+NFL) | $149/월. play-by-play·스탯·부상·오즈 전부 |
| TTS/STT | **ElevenLabs** / **Deepgram** | 음성 질문·답변용 |
| 크론 | **Hermes cronjobs** | 60초 간격 데이터 수집 + 질문 카드 생성 |
| 배포 | **Vercel** ($0 Hobby) 또는 **Hetzner $5** | |

## 10. 최종: 이 앱이 Google을 대체하는 이유

구글에 "Brunson stats tonight" 검색하는 건 **내가 뭘 모르는지 알고 있을 때**만 가능하다.

GamePulse는 **내가 뭘 모르는지 몰라도** 알려준다. 
- Wemby가 파울 트러블인지 몰랐어도 카드로 띄워줌
- NYK가 12-0 run 했는지 몰랐어도 타임라인에 표시됨
- Brunson이 커리어 하이 기록했는지 몰랐어도 푸시로 알려줌

**이게 StatMuse도, theScore도, Google도 못 하는 일이다.**
